# KOBO — Especificação de Marca e Site

> Documento-fonte para regenerar o sistema de marca Kobo (relatório, mockups e site) dentro do Claude Code ou qualquer agente com acesso a ferramentas de arquivo e geração de imagem.

---

## 1. Essência

- **Nome:** Kobo (工房, *kōbō* — atelier / oficina em japonês)
- **Tagline:** Móveis que brincam
- **Categoria:** Mobiliário infantil contemporâneo
- **Origem:** Brasil
- **Posicionamento:** Atelier brasileiro de mobiliário infantil em MDF, com encaixes à vista, bordas roliças e estética silenciosa. Peças lúdicas sem serem infantilizadas.
- **Valores:** honestidade material, montagem à vista como ornamento, durabilidade, design contido, segurança, brincar como ato projetual.
- **Tom de voz:** sóbrio, curto, sensorial, quase japonês. Evita diminutivos e superlativos.

## 2. Tipografia

- **Oficial única:** Maven Pro (Google Fonts)
- **Pesos:** 400 / 500 / 700 / 800 / 900
- **Uso:**
  - Logotipo: Maven Pro **800**, caixa alta, tracking `0.18em`
  - Títulos: Maven Pro **800** caixa alta, tracking `0.16em`–`0.32em`
  - Subtítulos / eyebrow: Maven Pro **500** caixa alta, tracking `0.28em`, cor Pedra
  - Corpo: Maven Pro **400**, caixa baixa
- **Regra absoluta:** nenhuma fonte serifada, nenhuma fonte de substituição.

## 4. Logo

- Marca palavra única: **KOBO**
- Assinatura opcional: ideograma **工房** posicionado como selo discreto (canto ou rodapé)
- Tagline sob a marca: `MÓVEIS QUE BRINCAM` em Maven Pro 500, tracking `0.28em`
- Área de respiro mínima: altura do "O" em todos os lados
- Proibições: gradientes, sombras, contornos, rotação, distorção, cores fora da paleta

## 7. Respondendo à pergunta: Claude Code + Flora AI

O Claude Code **não** tem integração direta e nativa com o Flora AI hoje. O que é possível:

- **Via MCP (Model Context Protocol):** se existir um servidor MCP para o Flora, basta adicionar ao `~/.claude.json` ou rodar `claude mcp add`. Até a data deste documento não há um MCP oficial publicado pelo Flora — vale checar em `https://flora.new` e no registro de MCPs.
- **Via API HTTP:** se o Flora expuser API REST/GraphQL, o Claude Code pode chamá-la diretamente com `curl`/`fetch` dentro de um script Node/Python que você peça para ele rodar. Você fornece a chave de API como variável de ambiente.
- **Via navegador (Claude in Chrome / Playwright MCP):** o agente pode abrir `flora.new`, fazer login **você mesmo**, colar o prompt e baixar a imagem. Funciona mas é frágil.
- **Alternativas com API pública estável e já prontas:** Replicate (SDXL, Flux), fal.ai, OpenAI Images, Stability AI, Ideogram. Qualquer uma dessas é chamável direto por `curl` no Claude Code sem MCP.

Recomendação prática: peça ao Claude Code para gerar as imagens via **Replicate + Flux** (custo baixo, sem login manual, resultado editorial excelente) usando os prompts da seção 6, salvar em `public/images/` e referenciar no site Next.js.

## 8. Ordem de execução sugerida

1. Gerar/confirmar `Kobo_Relatorio_de_Marca.pdf` (já existe em `mnt/outputs/`)
2. Gerar `Kobo_Brand_System.html` (já existe)
3. Rodar `npx create-next-app@latest kobo-site --ts --tailwind --app`
4. Aplicar tokens de cor e importar Maven Pro em `app/layout.tsx`
5. Gerar imagens via API de imagem escolhida → `public/images/`
6. Montar Home, Coleção, Atelier, Contato
7. Deploy em Vercel
